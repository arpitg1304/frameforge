The problem

You have 10,000 manipulation episode MP4s and need a PyTorch DataLoader that samples clips within episode boundaries, works with num_workers > 0 without crashing, and doesn't require extracting every frame to JPEG (2 TB as video → 18 TB as images).

Every robotics lab writes 200 lines of custom code for this. frameforge is those 200 lines, tested and reusable.

How it works

Episode-aware DataLoader

EpisodeSampler never crosses task boundaries. Readers open inside __getitem__, not __init__ — no forked file handle bugs. Works with any number of workers. 

from frameforge import VideoReader
from frameforge.sampling import EpisodeSampler
from frameforge.dataloader import VideoClipDataset

dataset = VideoClipDataset(
    video_paths=episode_files,
    sampler=EpisodeSampler(
        boundaries=[0, 3000, 6000],
        num_frames=16),
    backend="pyav",
)

# This won't crash.
loader = DataLoader(
    dataset, batch_size=32,
    num_workers=8)

Multi-camera sync

Wrist, overhead, third-person — get time-aligned frames from all cameras in one call. 

from frameforge.sampling.sync import (
    sync_readers_by_timestamp,
)

readers = {
    "wrist": VideoReader("wrist.mp4"),
    "overhead": VideoReader("overhead.mp4"),
}

synced = sync_readers_by_timestamp(
    readers,
    timestamps_sec=[0.0, 0.5, 1.0],
)
# synced["wrist"]    → (3, H, W, C)
# synced["overhead"] → (3, H, W, C)

Read from any backend

Swap between PyAV, torchcodec, decord, or OpenCV with one argument. Output tensor format is always the same. 

# Auto-select best available
reader = VideoReader("episode.mp4")

# Or pick explicitly
reader = VideoReader("episode.mp4",
    backend="torchcodec",
    device="cuda:0")

# Pythonic indexing
frame = reader[0]             # (H, W, C)
clip  = reader[100:116]       # (16, H, W, C)
picks = reader[[0, 50, 100]] # (3, H, W, C)

Who this is for

Teams training manipulation policies (ACT, Diffusion Policy, VLAs) from episode MP4s. Multi-camera rigs that need synchronized frame reads. Anyone with existing video datasets who doesn't want to extract frames or adopt a full platform just to load clips.
